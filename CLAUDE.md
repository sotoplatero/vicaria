## Project Overview

This is a SvelteKit 2 application built with:
- **Svelte 5** (using runes syntax)
- **TypeScript** with strict mode enabled
- **Tailwind CSS 4** via Vite plugin
- **DaisyUI** for component styling
- **mdsvex** for Markdown in Svelte components (.svx files)
- **Paraglide JS** for i18n (internationalization)

## Package Manager

This project uses **pnpm**. Always use `pnpm install` for dependencies, not npm or yarn.

## Development Commands

- `pnpm dev` - Start development server
- `pnpm build` - Create production build
- `pnpm preview` - Preview production build locally
- `pnpm check` - Run TypeScript and Svelte checks
- `pnpm check:watch` - Run checks in watch mode

## Architecture

### Internationalization (i18n)

The app uses Paraglide JS for i18n with the following setup:

- **Base locale**: English (`en`)
- **Supported locales**: `en`, `es`
- **Message files**: `messages/{locale}.json`
- **Generated code**: `src/lib/paraglide/` (auto-generated by Vite plugin)
- **Configuration**: `project.inlang/settings.json`

**Key i18n files:**
- `src/hooks.server.ts:4-10` - Server-side middleware that injects locale into HTML via `%paraglide.lang%` placeholder
- `src/hooks.ts:3` - Client-side URL rerouting using `deLocalizeUrl()` to strip locale prefixes from URLs

When adding new translatable strings, update the JSON files in `messages/` directory. The Paraglide Vite plugin will auto-generate the necessary code.

### File Extensions

The project accepts both `.svelte` and `.svx` (Markdown + Svelte) files due to mdsvex preprocessor configuration in `svelte.config.js:14`.

### Styling

- **Tailwind CSS 4**: Configured via `@tailwindcss/vite` plugin, no separate config file needed
- **DaisyUI**: Available for pre-built component classes
- **Typography plugin**: `@tailwindcss/typography` for prose styling
- Global styles in `src/app.css`

### SvelteKit Structure

Standard SvelteKit conventions apply:
- Routes in `src/routes/`
- Shared layout at `src/routes/+layout.svelte`
- Static assets in `static/`
- Library code in `src/lib/`
- Type definitions in `src/app.d.ts`

## Important Notes

- The model configuration (gpt-4.1-mini) should not be changed
- Svelte 5 uses runes (`$state`, `$derived`, `$props`, etc.) - do not use Svelte 4 patterns
- TypeScript is in strict mode with `checkJs` enabled - maintain type safety
